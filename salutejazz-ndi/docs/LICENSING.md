# Лицензионные условия SaluteJazz SDK

Результаты исследования по состоянию на май 2026.

## Что официально известно

### SDK Key
- Для использования `@salutejs/jazz-sdk-web` (и других SDK семейства) **обязателен SDK Key**.
- Ключ выдаётся через личный кабинет на `developers.sber.ru` (раздел "Лицензии"
  / "Управление ключами").
- В терминах Sber это называется "SDK Key" (содержит `projectId` + ECDSA-приватный
  ключ в JWK-формате, закодированный в base64).
- На основе SDK Key клиентское приложение генерирует **JWT-транспортный токен**
  (подпись ECDSA по P-256/P-384/P-521), который SDK обменивает на
  **Jazz access token** через `POST /v1/auth/login`.
- Срок жизни транспортного токена — 120 секунд по умолчанию.
- **Рекомендация Sber**: транспортный токен генерировать на сервере (бэкенд),
  чтобы SDK Key не утёк в публичный код. В headless-сценарии (наше приложение
  работает только локально как операторский тул) можно держать SDK Key в
  локальной зашифрованной конфигурации.

### Версия SDK Demo
- Демо-проект `salute-developers/jazz-web-sdk-demo` опубликован под MIT.
- Версии npm-пакетов на момент клонирования: `26.11.23` (`@salutejs/jazz-sdk-web`,
  `@salutejs/jazz-sdk-web-plugins`, `@salutejs/jazz-sdk-electron`,
  `@salutejs/jazz-sdk-electron-plugins`).
- Сами npm-пакеты SDK на npmjs.com скрыты от прямого WebFetch (403), но
  публикуются под `@salutejs/...` и подключаются демо-проектом штатно.

### Тарифы и лимиты
- **Бесплатная версия SaluteJazz** (без коммерческого договора):
  - до 100 участников в комнате, до 60 минут на встречу, веб + десктоп клиенты.
- **Бизнес-тарифы** (Jazz Meetings, Jazz Webinars):
  - до 600 одновременных участников (Meetings).
  - до 3000 участников и 100 спикеров (Webinars).
- **On-premise** доступен от 100 лицензий, обсуждается индивидуально с Sber.
- Цены на API/SDK на публичной странице **не указаны** — Sber работает по
  договорным условиям. Бесплатная разработка (получение тестового SDK Key для
  внутренних интеграций) **возможна**, но для production требуется договор.

### Сценарий "подключение к чужой комнате"
Этот вопрос — самый чувствительный для нашей задачи (вещание из любой
встречи SaluteJazz, а не только своей).

**Что подтверждено документацией**:
- `client.conferences.join({roomId, password}, withAnonymousJoin({userName}))` —
  стандартный сценарий присоединения к существующей комнате анонимным
  пользователем по `roomId` и паролю.
- В `JoinToConferenceModal` демо-проекта пользователь вводит roomId+password —
  SDK подключается к комнате без её предварительного создания через API.

**Что неоднозначно**:
- Привязан ли SDK Key к домену организации, и есть ли у Sber whitelist по
  `serverUrl` — публично не описано.
- При вызове `createJazzClient(sdk, {serverUrl})` в демо-проекте host —
  поле, которое пользователь вводит свободно. То есть в принципе можно
  направить клиент на `https://salutejazz.ru` (публичный сервер) или на
  on-prem инстанс компании. Лимиты на стороне сервера на анонимный join —
  открытый вопрос.

**Практически**: для вещания **в свои корпоративные конференции** проблем нет.
Для вещания **в публичные/чужие** — нужно либо получить разрешение
организатора (доступ по ссылке + пароль и ваш SDK Key, выпущенный Sber),
либо выяснять условия лицензии напрямую у Sber sales.

## Что важно проверить **до подписания договора**

1. **Разрешено ли в публичной оферте/договоре** использовать SDK для
   автоматического (headless) присоединения к встречам и форвардинга медиа в
   третьи системы (NDI). Это нестандартный use case — может быть в серой зоне.
2. **Лимит одновременных подключений** на один SDK Key (большие конференции
   требуют много одновременных join'ов от одного клиента).
3. **Качество видео**: бесплатная версия может сжимать до 720p, в Pro/Business —
   1080p. NDI-вещание должно идти на 1080p минимум.
4. **Гарантия SLA** на серверы SaluteJazz при on-prem или cloud.

## Рекомендация по запуску

Для **MVP/прототипа**:
- Получить тестовый SDK Key через `developers.sber.ru` (бесплатно, для
  разработки).
- Использовать его в нашем приложении, проверить полный пайплайн на 1-3
  участников.
- Написать в Sber sales про коммерческое использование с указанием use-case
  ("вещание участников из встречи в NDI для продакшн-режиссуры").

Для **production**:
- Подписать договор Enterprise.
- Если требуется автономность — рассматривать **on-premise SaluteJazz**:
  внутри корпоративного контура наш плагин получает прямой доступ к серверу
  без лицензионных ограничений на cloud-инстанс.
